Map

Map key sifatida hamma data typeni qabul qiladi

Unda Garbage collection ishlamaydi.

Map metodlari

  • new Map() – map yaratadi.
  • map.set(key, value) – mapga key va value qo'shadi.
  • map.get(key) – key orqali valueni chiqaradi agar mavjud bo'lmagan key kiritilsa undefined qaytaradi
  • map.has(key) – berilgan key map ichida bor yo'qligiga qarab true va false qaytaradi
  • map.delete(key) – key kritilsa map ichidagi property ni o'chirib yuboradi
  • map.clear() - mapni tozalaydi
  • map.size - mapni lengthini chiqarib beradi

WeakMap

WeakMap metodlari

  • weakMap.set(key, value) - unga key va value qo'shadi

  • weakMap.delete(key) - weakMapni tozalaydi

Set

Set unique qiymat olish uchun ishlatiladi va faqat valuelardan iborat bo'ladi unda ham garbage collection ishlamaydi chunki unda key yo'q

  • new Set([iterable]) - set yaratish
  • set.add(value) - value qo'shadi
  • set.delete(value) - valueni olib tashlaydi va agar o'sha paytda value kiritilsa true aks holda false qaytaradi
  • set.has(value) - value ning bor yoki yo'qligiga qarab true||false qaytaradi
  • set.clear() - setni tozalaydi
  • set.size - setning lengthini aniqlab beradi

Property Flags

Property Flags 3 ga bo'linadi

1. writable

Objectni qaysidir propertysini writable i true bo'lsa o'zgartirib(overwrite) bo'ladi false bo'lsa o'zgartirib bo'lmaydi

2. enumerable

Objectni qaysidir propertysini enumerable ini true qilinsa bu property loopda ko'rinadi agar false qiladigan bo'lsak u loopda ko'rinmaydi

3. configurable

Objectni qaysidir propertysini configurable true bo'lsa delete qilib bo'ladi agar false qilinsa o'sha propertyni delete qilib bo'ladi

Agar bitta propertyni o'zgartirmoqchi bo'lsak Object.defineProperty(object name, 'property key', {property flag: true or false }) dan foydalaniladi

"property key:" { property flag: true or false } })

Functional Programming

Functional Programming declarative programming paradigmlar sinfiga mansub bo'lib, dasturni pure functionlar orqali ishlatishni ilgari suradi. BU orqali kod tushunarli, boshqarishga oson, qayta ishlatsa bo'ladigan va testlashga oson bo'ladi. Functional programmingda quyidagi conceptsiyalar mavjud: Immutability, Pure Function, Higher ordered functions, first-class function, Currying, Recursion, Composition

Immutability

Immutability - bu dasturdagi data (object, array ...) larni aslini o'zgartirmasdan ular bila ishlash. Bunda arraylar bilan ishlaganda mutable methodlardan iloji boricha qochiladi. Masalan: sort(), reverse() lar o'rniga map(), filter(), reduce() kabi methodlar ishlatiladi

Pure function

Pure Function tashqi tomondagi variablega bog'liq bo'lmaydi va argumentga bog'liq bo'lib qiymat qaytaradi va side effect yo'q

return a + b

}

pureFn(a, b)

OOP va uning prinsiplari

OOP - bu dasturdagi kodlarni vazifasiga qarab, objectlar orqali turli qismlarga ajratib, kodlarni takrorlanishini oldini olib, qayta ishlatish imkonini beradigan - PROGRAMMING PARADIGM Uning 4ta asosiy prinsiplari mavjud:

1. Inheritance - bir Class ikkinchi Classga meros qoldirish yoki Classdan ikkinchi Classda foydalanish(extends, super)

2. Encapsulation - Class tashqariga o'zidagi nimani ko'rsatish yoki ko'rsatmasligiga aytiladi. Ya'ni public, private, protected bo'lgan method yoki propertylar orqali qilinadi gan amallar Encapsulation qilishga kiradi (getter/setter) lar ham.

3. Abstraction - bu Class faqat ishlatilinadigan method/propertylarni tashqariga chiqaradi, keraksizlarni o'z ichiga qoldirish ya'ni yashirish

4. Polimorphism - methodlar sharoitga qarab har xil qiymat qaytarishi. Masalan satHello() har bir userda har xil ishlaydi. Method qaterda ishlatilsa shu yerga qarab ishlaydi constructorga berilgan qiymatga bog'liq bo'ladi.
